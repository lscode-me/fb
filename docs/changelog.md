# Что нового

История изменений книги «Файлы: missing manual».

## Март 2026

### 06.03.2026

- **Ch5/36 UNIX ФС**: масштабное расширение главы
  - Эволюционная диаграмма ФС расширена с 9 до 23 файловых систем, добавлены семейства (BSD, Linux, Solaris)
  - Исправлено описание связи FFS→UFS: отдельная врезка с историей 4.2BSD→SunOS→SVR4
  - Добавлена таблица «Операционная система → файловая система по умолчанию»
  - ext2→ext3: добавлен tip о конвертации через `tune2fs -j`
  - ext4: расширено описание технологий (extents, multiblock allocation, delayed allocation)
  - XFS: добавлена история (SGI IRIX → Linux) и архитектура (allocation groups, B+ trees)
  - Добавлен пояснительный текст к разделам монтирования, fsck и VFS
  - Расширена итоговая таблица ФС до 12 файловых систем
- **Ch5/34 Архитектура ФС**: добавлен раздел «Сколько дисковых операций стоит одно чтение?»
  - Подсчёт I/O для `open()` + `read()`: ~8 дисковых чтений, ~5 записей при создании файла
  - Роль page cache, групп блоков и буферизации записи
- **Ch5/36 UNIX ФС**: расширены разделы о crash consistency и журналировании
  - Таблица 6 crash-сценариев: почему без журнала 5 из 6 повреждают ФС
  - Протокол write-ahead logging (TxBegin → journal → TxEnd → checkpoint)
  - Проблема write barriers и переупорядочивание записей диском
  - Политика размещения больших файлов в FFS (large-file exception, амортизация seek)
  - Раздел «Чего не решают классические ФС: silent data corruption» в резюме
- **Ch5/40 ZFS**: расширена мотивация — silent data corruption как причина создания ZFS
  - Статистика: 0.5% SATA-дисков имеют silent corruption за 3 года, 9.4% — LSE
- **Ch5/41 Btrfs**: добавлена мотивация — тихое повреждение данных и как Btrfs решает это
  - CoW + checksums + RAID self-healing

### 03.03.2026

- **Ch4/27 Coreutils**: добавлен практический пример бинарного патчинга с `dd` и `grep`
  - Поиск оффсета в бинарном файле и замена байтов
  - Альтернативное решение на Python для сравнения подходов
- **Ch5/36 XFS**: добавлена информация про `xfs_healer` (Linux 7.0)
  - Автоматическое обнаружение и восстановление повреждений XFS в online-режиме
  - **Дополнено**: требования форматирования (`rmapbt=1`, `parent=1`), настройка `autofsck`, kernel 6.12+, экспериментальный статус
  - **Дополнено**: детали работы через `XFS_IOC_HEALTH_MONITOR` ioctl, типы событий мониторинга
- **Ch5/39 Linux ФС**: новый раздел 39.6 «Read-Only файловые системы»
  - EROFS с поддержкой shared page cache (Linux 7.0+)
  - **Дополнено**: механизм fingerprints в extended attributes (`trusted.erofs.fingerprint`)
  - **Дополнено**: опции монтирования (`inode_share`, `domain_id`), требования безопасности
  - **Дополнено**: ограничения (несовместимость с direct I/O)
  - SquashFS, ISO9660, cramfs — сравнение и рекомендации
- **Ch5/49 Контейнеры**: расширен раздел про альтернативы OverlayFS
  - EROFS для контейнерных образов
  - Shared page cache для дедупликации памяти между контейнерами
  - **Дополнено**: примеры монтирования с `inode_share` и `domain_id`
- **Глоссарий**: обновлены определения `xfs_healer`, добавлены термины: `cramfs`, `dd`, `EROFS`, `ISO9660`, `OverlayFS`, `SquashFS`
- **Исправления**: выравнивание ASCII-схем в главе 49
