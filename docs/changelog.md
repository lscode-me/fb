# Что нового

История изменений книги «Файлы: missing manual».

## Март 2026

### 03.03.2026

- **Ch4/27 Coreutils**: добавлен практический пример бинарного патчинга с `dd` и `grep`
  - Поиск оффсета в бинарном файле и замена байтов
  - Альтернативное решение на Python для сравнения подходов
- **Ch5/36 XFS**: добавлена информация про `xfs_healer` (Linux 7.0)
  - Автоматическое обнаружение и восстановление повреждений XFS в online-режиме
  - **Дополнено**: требования форматирования (`rmapbt=1`, `parent=1`), настройка `autofsck`, kernel 6.12+, экспериментальный статус
  - **Дополнено**: детали работы через `XFS_IOC_HEALTH_MONITOR` ioctl, типы событий мониторинга
- **Ch5/39 Linux ФС**: новый раздел 39.6 «Read-Only файловые системы»
  - EROFS с поддержкой shared page cache (Linux 7.0+)
  - **Дополнено**: механизм fingerprints в extended attributes (`trusted.erofs.fingerprint`)
  - **Дополнено**: опции монтирования (`inode_share`, `domain_id`), требования безопасности
  - **Дополнено**: ограничения (несовместимость с direct I/O)
  - SquashFS, ISO9660, cramfs — сравнение и рекомендации
- **Ch5/49 Контейнеры**: расширен раздел про альтернативы OverlayFS
  - EROFS для контейнерных образов
  - Shared page cache для дедупликации памяти между контейнерами
  - **Дополнено**: примеры монтирования с `inode_share` и `domain_id`
- **Глоссарий**: обновлены определения `xfs_healer`, добавлены термины: `cramfs`, `dd`, `EROFS`, `ISO9660`, `OverlayFS`, `SquashFS`
- **Исправления**: выравнивание ASCII-схем в главе 49

### 01.03.2026

- **Ch0, Ch1**: добавлена история файлов — от перфокарт и магнитных лент до Unix и современных ФС
- **Ch2/11 Байты**: добавлен раздел о байте как минимальной единице данных в файле
- **Ch3/24 Архивы**: добавлен раздел о принципах работы алгоритмов сжатия (Huffman, LZ77, DEFLATE)
- **Ch5/36 UNIX ФС**: расширено описание archivemount как практического примера FUSE-ФС для работы с архивами
- **Техническое**: robots.txt — блокировка `/cdn-cgi/` от индексации
