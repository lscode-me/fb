  - Добавлены инструменты и стандарты: fsck, Docker/OCI, POSIX, procfs/sysfs, tokenization
  - Исправлен алфавитный порядок внутри секций; добавлена секция W; расширены определения XFS, ext4, NTFS
- **Ch5/36 UNIX ФС**: масштабное расширение главы
  - Эволюционная диаграмма ФС расширена с 9 до 23 файловых систем, добавлены семейства (BSD, Linux, Solaris)
  - Исправлено описание связи FFS→UFS: отдельная врезка с историей 4.2BSD→SunOS→SVR4
  - Добавлена таблица «Операционная система → файловая система по умолчанию»
  - ext2→ext3: добавлен tip о конвертации через `tune2fs -j`

  - XFS: добавлена история (SGI IRIX → Linux) и архитектура (allocation groups, B+ trees)
  - Добавлен пояснительный текст к разделам монтирования, fsck и VFS
  - Расширена итоговая таблица ФС до 12 файловых систем

  - Подсчёт I/O для `open()` + `read()`: ~8 дисковых чтений, ~5 записей при создании файла
  - Роль page cache, групп блоков и буферизации записи
- **Ch5/36 UNIX ФС**: расширены разделы о crash consistency и журналировании
  - Таблица 6 crash-сценариев: почему без журнала 5 из 6 повреждают ФС
  - Протокол write-ahead logging (TxBegin → journal → TxEnd → checkpoint)
  - Проблема write barriers и переупорядочивание записей диском
  - Политика размещения больших файлов в FFS (large-file exception, амортизация seek)
  - Раздел «Чего не решают классические ФС: silent data corruption» в резюме
- **Ch5/40 ZFS**: расширена мотивация — silent data corruption как причина создания ZFS
