# Глоссарий

Ключевые термины книги. При первом упоминании в тексте термин даётся с определением; здесь — полный алфавитный справочник.

---

## A

ACL (Access Control List)
:   Список управления доступом. Расширенная модель прав, позволяющая задавать разрешения для произвольных пользователей и групп, в отличие от базовой модели owner/group/other. → Глава 9

ASCII (American Standard Code for Information Interchange)
:   7-битная кодировка символов, определяющая 128 символов (33 управляющих + 95 печатных). Основа всех современных кодировок. → Глава 12

Avro
:   Строковый формат сериализации данных Apache. Хранит схему вместе с данными, используется в Hadoop-экосистеме. → Глава 23

## B

Base64
:   Схема кодирования бинарных данных в ASCII-символы. Используется для передачи бинарных данных через текстовые протоколы (email, JSON). → Глава 11

Big-endian / Little-endian
:   Порядок байтов (byte order, endianness). Big-endian — старший байт первый (сетевой порядок). Little-endian — младший байт первый (x86). → Глава 10

Block device (блочное устройство)
:   Устройство хранения, предоставляющее доступ к данным блоками фиксированного размера (обычно 512 байт или 4 КБ). Примеры: `/dev/sda`, `/dev/nvme0n1`. → Глава 34

BOM (Byte Order Mark)
:   Специальный символ Unicode U+FEFF в начале файла, указывающий порядок байтов для UTF-16/UTF-32. В UTF-8 не требуется, но иногда добавляется Windows-приложениями. → Глава 14

Btrfs (B-tree File System)
:   Файловая система Linux с поддержкой copy-on-write, снапшотов, встроенного RAID, сжатия и дедупликации. → Глава 41

## C

Codec (кодек)
:   Алгоритм сжатия/распаковки данных. В контексте медиа: H.264 (видео), AAC (аудио), Opus (аудио). Не путать с контейнером. → Глава 25

Container (контейнер, формат-контейнер)
:   Файловый формат, объединяющий несколько потоков данных (видео, аудио, субтитры). Примеры: MP4, MKV, WebM. Не путать с Docker-контейнерами. → Глава 25

Copy-on-Write (CoW)
:   Стратегия записи: данные не перезаписываются на месте, а записываются в новое место. Используется в ZFS, Btrfs. Позволяет создавать мгновенные снапшоты. → Главы 40, 41

Coreutils
:   Набор базовых утилит GNU/POSIX: `cat`, `head`, `tail`, `sort`, `uniq`, `cut`, `tr`, `wc`, `tee`, `xargs` и др. → Глава 27

CSV (Comma-Separated Values)
:   Текстовый табличный формат. Каждая строка — запись, поля разделены запятыми. Стандарт: RFC 4180. → Глава 19

## D

Dentry (directory entry)
:   Запись в директории, связывающая имя файла с номером inode. Директория — это файл, содержащий список dentry. → Глава 2

DuckDB
:   Встраиваемая аналитическая СУБД (OLAP). Позволяет выполнять SQL-запросы напрямую к CSV, Parquet, JSON файлам без импорта. → Глава 33

## E

ELF (Executable and Linkable Format)
:   Формат исполняемых файлов в Linux/Unix. Содержит заголовок, секции кода и данных, таблицу символов. Magic number: `\x7fELF`. → Глава 22

Endianness
:   См. Big-endian / Little-endian.

ext4 (Fourth Extended Filesystem)
:   Основная файловая система Linux. Журналируемая, поддерживает файлы до 16 ТБ, разделы до 1 ЭБ. → Глава 39

## F

fd (file descriptor, файловый дескриптор)
:   Целочисленный идентификатор открытого файла в процессе. 0 = stdin, 1 = stdout, 2 = stderr. → Глава 7

FIFO (named pipe, именованный канал)
:   Специальный файл для межпроцессного взаимодействия. Создаётся через `mkfifo`. Данные передаются в порядке FIFO. → Глава 26

FUSE (Filesystem in Userspace)
:   Интерфейс для создания файловых систем в пользовательском пространстве без модификации ядра. Примеры: sshfs, s3fs. → Глава 34

## G

GPT (GUID Partition Table)
:   Современная схема разметки дисков, замена MBR. Поддерживает до 128 разделов и диски > 2 ТБ. → Глава 35

## H

Hard link (жёсткая ссылка)
:   Дополнительное имя (dentry) для существующего inode. Все жёсткие ссылки равноправны. Файл удаляется когда счётчик ссылок достигает 0. → Глава 4

## I

Inode (index node)
:   Структура данных в файловой системе, хранящая метаданные файла: размер, владелец, права, временные метки, указатели на блоки данных. Не содержит имя файла. → Глава 3

IOPS (Input/Output Operations Per Second)
:   Метрика производительности накопителя — количество операций ввода-вывода в секунду. HDD: ~100–200, SSD: ~10 000–100 000, NVMe: ~100 000–1 000 000. → Глава 34

## J

JSON (JavaScript Object Notation)
:   Текстовый формат обмена данными. Поддерживает: объекты, массивы, строки, числа, boolean, null. → Глава 16

JSON Lines (JSONL)
:   Формат, где каждая строка файла — отдельный JSON-объект. Удобен для потоковой обработки и логов. → Глава 16

Journaling (журналирование)
:   Техника обеспечения целостности ФС: операции сначала записываются в журнал, затем применяются. При сбое — повтор или откат из журнала. → Главы 36, 39

## K

KOI8-R
:   8-битная кодировка кириллицы. Особенность: при сбросе 8-го бита буквы дают читаемую транслитерацию. → Глава 13

## L

LVM (Logical Volume Manager)
:   Менеджер логических томов Linux. Абстракция между физическими дисками и файловыми системами. Позволяет расширять/уменьшать тома, создавать снапшоты. → Глава 43

## M

Mach-O (Mach Object)
:   Формат исполняемых файлов macOS/iOS. Аналог ELF для Apple-платформ. → Глава 22

Magic number (магическое число)
:   Последовательность байтов в начале файла, идентифицирующая формат. Примеры: `\x89PNG` (PNG), `%PDF` (PDF), `PK` (ZIP). → Глава 22

MBR (Master Boot Record)
:   Устаревшая схема разметки дисков. Ограничения: максимум 4 основных раздела, максимум 2 ТБ. → Глава 35

MessagePack
:   Бинарный формат сериализации, совместимый по модели данных с JSON, но компактнее и быстрее. → Глава 21

mmap (memory-mapped file)
:   Механизм отображения файла в память процесса. Позволяет работать с файлом как с массивом байтов. → Глава 31

## N

NFC / NFD (Unicode Normalization)
:   Формы нормализации Unicode. NFC — composed (é = один code point). NFD — decomposed (é = e + combining accent). Важно при сравнении строк и именах файлов (macOS использует NFD). → Глава 14

NTFS (New Technology File System)
:   Файловая система Windows. Поддерживает журналирование, ACL, ADS (Alternate Data Streams), сжатие, шифрование. → Глава 37

## O

Object storage (объектное хранилище)
:   Архитектура хранения данных без иерархии каталогов. Каждый объект = ключ + данные + метаданные. Пример: Amazon S3. → Глава 45

## P

Parquet
:   Колоночный бинарный формат для аналитических данных. Эффективное сжатие, предикатный pushdown. Стандарт де-факто в Big Data. → Глава 23

PE (Portable Executable)
:   Формат исполняемых файлов Windows (`.exe`, `.dll`). → Глава 22

Pipe (канал, пайп)
:   Механизм соединения stdout одного процесса со stdin другого: `cmd1 | cmd2`. Основа философии UNIX. → Глава 26

Protocol Buffers (Protobuf)
:   Бинарный формат сериализации от Google. Требует определения схемы (`.proto` файл). Компактный и быстрый. → Глава 21

## R

RAID (Redundant Array of Independent Disks)
:   Технология объединения дисков для повышения надёжности и/или производительности. Уровни: 0, 1, 5, 6, 10. → Глава 42

## S

S3 (Simple Storage Service)
:   Объектное хранилище Amazon. API стал стандартом де-факто (MinIO, Ceph). Ключевые понятия: bucket, object, presigned URL. → Глава 45

Sparse file (разреженный файл)
:   Файл, в котором последовательности нулевых байтов не занимают место на диске. `ls -l` показывает логический размер, `du` — фактический. → Глава 3

Symbolic link (симлинк, символическая ссылка)
:   Специальный файл, содержащий путь к другому файлу. В отличие от жёсткой ссылки, может указывать на файлы в других ФС и на несуществующие файлы. → Глава 4

## T

TOML (Tom's Obvious Minimal Language)
:   Текстовый формат конфигураций. Явная типизация (даты, числа). Используется в `pyproject.toml`, `Cargo.toml`. → Глава 18

## U

umask
:   Маска прав по умолчанию для новых файлов. Определяет, какие биты прав **не** устанавливаются. Типичное значение: 022 (файлы создаются с 644). → Глава 6

Unicode
:   Стандарт кодирования символов всех письменностей мира. Определяет ~150 000 символов в 17 плоскостях (planes). Code point записывается как U+XXXX. → Глава 14

UTF-8
:   Кодировка Unicode с переменной длиной (1–4 байта на символ). ASCII-совместима. Стандарт де-факто для web и Unix. → Глава 14

## V

VFS (Virtual File System)
:   Абстракционный слой ядра, предоставляющий единый интерфейс для разных файловых систем. Программа вызывает `open()` — VFS направляет вызов в нужный драйвер ФС. → Глава 34

## X

xattr (extended attributes)
:   Расширенные атрибуты файлов — пары ключ-значение, хранимые в inode. Используются для SELinux-меток, capabilities, пользовательских данных (`user.*`). → Глава 9

XFS
:   Высокопроизводительная 64-битная журналируемая ФС. Оптимальна для больших файлов и параллельного I/O. Файловая система по умолчанию в RHEL/CentOS. → Глава 39

XML (Extensible Markup Language)
:   Текстовый формат для структурированных данных с тегами. Поддерживает пространства имён, схемы (DTD, XSD), запросы (XPath). → Глава 20

## Y

YAML (YAML Ain't Markup Language)
:   Текстовый формат конфигураций, использующий отступы для вложенности. Надмножество JSON. Широко используется в Kubernetes, Ansible, CI/CD. → Глава 17

## Z

ZFS (Zettabyte File System)
:   Файловая система и менеджер томов. Объединяет ФС, RAID (RAID-Z), LVM в единую архитектуру. Поддерживает CoW, снапшоты, дедупликацию, сжатие, self-healing. → Глава 40

zstd (Zstandard)
:   Алгоритм сжатия от Facebook. Компромисс между скоростью (как LZ4) и степенью сжатия (как zlib). Настраиваемый уровень 1–22. → Глава 24
